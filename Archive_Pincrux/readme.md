# AI_Agent 시스템 운영 가이드

## 프로젝트 목표

슬랙(Slack)에서 매일 발생하는 비즈니스 대화를 자동으로 수집·분류하여, 컨플루언스(Confluence) 위키에 구조화된 문서로 누적 관리하는 4단계 AI 에이전트 파이프라인이다.

사람이 직접 슬랙 대화를 읽고 위키를 업데이트하는 반복 작업을 자동화함으로써, COO(총총)가 전체 비즈니스 컨텍스트를 위키 단일 진실 공급원(Single Source of Truth)으로 유지할 수 있게 한다.

---

## 파이프라인 흐름

```
슬랙 원본 대화
      ↓
[01 모으미] 수집·분류 → JSON
      ↓
[02 쓰미] 컨플루언스 위키 병합/생성
      ↓
[03 읽어봄이] 교차 검증 (PASS / FAIL)
      ↓ PASS
[04 다시쓰미] 문체 윤문·마크다운 최적화 → 최종 퍼블리싱
      ↓ FAIL
    총총(COO) 개입 → 쓰미 재작업 지시
```

## 커넥터

| 커넥터 | 접근 노드 | 용도 |
|--------|-----------|------|
| **슬랙(Slack)** | 모으미(v1) 전용 | 원시 대화 로그 수집 |
| **컨플루언스(Confluence) — 스페이스: Archive_Pincrux (AR)** | 쓰미(v2), 읽어봄이(v3), 다시쓰미(v4) | 위키 생성·병합·검증·최종 퍼블리싱 |

> **단일 진실 공급원**: 쓰미가 컨플루언스 AR 스페이스에 최초 업로드한 시점부터 해당 컨플루언스 페이지가 원본이다. 로컬 JSON 파일은 파이프라인 이력 보관용이며 원본이 아니다.

## 분류 체계

**1단계 — 주체(Major Category)**
- `Client`: 외부 비용을 받고 대행하는 광고주
- `Media_Partner`: 광고 인벤토리 거래 및 사업 제휴 관계사
- `In-house`: 자사 비용·인력이 투입되는 내부 과제 및 R&D

**2단계 — 업무 형태(Minor Category)**
- `Project`: 기한이 정해진 단발성 업무 (캠페인, 외부 구축 등)
- `Product`: 지속 고도화하는 자산형 서비스 (플랫폼, 솔루션 등)

**3단계 — 맥락 정합성(Context Status)**
- `matched`: 기존 진행 중인 프로젝트/프로덕트의 후속 업데이트
- `new`: 기존 분류 체계에 없던 신규 주체·캠페인·기획
- `inquiry`: 단순 버그 제보, 내부 문의, 단발성 Q&A
- `unknown`: 파편화된 대화로 맥락 판별 불가 → 총총 개입 대기

---

## 운영 원칙

1. **단일 진실 공급원**: 모든 노드의 행동 규칙은 `directive_hub/`의 `.md` 파일이 유일한 기준이다.
2. **COO 중앙집권**: 하위 노드의 지침 수정은 반드시 총총(COO)을 통한다. 노드 간 임의 수정 금지.
3. **Fail-Safe**: 읽어봄이가 FAIL을 반환하면 위키 최종 반영을 중단하고 총총이 개입한다.
4. **보안**: API 키 등 민감 정보는 지침서 파일에 절대 기재하지 않는다.
