# 파이프라인 데이터 교환 인터페이스 규격 — 색인

> **2026-07-20 개편**: 이 문서는 원래 공통 규칙 + 4개 노드 스키마를 한 파일(약 8,200자)에 모두 담고 있었으나, 각 노드가 실제로 필요한 부분보다 훨씬 많은 분량을 매번 읽어야 하는 비효율이 있어 아래와 같이 분리했다. **이 파일 자체는 더 이상 어떤 노드도 전체를 읽을 필요가 없다** — 아래 색인에서 자신에게 필요한 파일만 읽는다.

| 파일 | 내용 | 필요한 노드 |
|---|---|---|
| `interface_common.md` | 전체 데이터 흐름, 파일 경로 규칙, 공통 상태 코드, 에러 포맷, 백필 체크포인트 | 전 노드 공통 |
| `interface_schema_collector.md` | 4-1 모으미 출력 스키마 | 모으미(자기 출력), 쓰미·읽어봄이(입력 이해) |
| `interface_schema_writer.md` | 4-2 쓰미 출력 스키마 | 쓰미(자기 출력), 읽어봄이(입력 이해) |
| `interface_schema_validator.md` | 4-3 읽어봄이 출력 스키마 | 읽어봄이(자기 출력), 다시쓰미(입력 이해) |
| `interface_schema_refiner.md` | 4-4 다시쓰미 출력 스키마 | 다시쓰미(자기 출력) |

각 노드가 실제로 읽어야 할 파일 조합은 해당 노드의 지침서(v1~v4)에 명시되어 있다.
